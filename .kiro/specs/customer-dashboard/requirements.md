# Requirements Document

## Introduction

The Customer Dashboard provides users with a comprehensive overview of their article generation system. It displays key metrics about articles and subjects, allows users to view and manage their content queue, and provides real-time updates as data changes. The dashboard serves as the central hub for users to monitor their article generation progress and manage their content strategy.

## Glossary

- **Dashboard**: The main user interface displaying aggregated metrics and content overview
- **Article**: A piece of content generated by the AI system based on user-defined subjects
- **Subject**: A topic or theme that the user wants articles generated about
- **Metrics**: Quantitative measurements of article generation activity (counts, statuses)
- **Topics Queue**: The list of subjects waiting to be processed for article generation
- **Delivery Days**: Days of the week when articles are scheduled for delivery

## Requirements

### Requirement 1

**User Story:** As a user, I want to see an overview of my article generation metrics, so that I can track my content production progress.

#### Acceptance Criteria

1. WHEN a user navigates to the dashboard THEN the Dashboard SHALL display the total count of articles generated
2. WHEN a user navigates to the dashboard THEN the Dashboard SHALL display the count of articles sent via email
3. WHEN a user navigates to the dashboard THEN the Dashboard SHALL display the count of articles in draft status
4. WHEN a user navigates to the dashboard THEN the Dashboard SHALL display the count of subjects in the topics queue
5. WHEN metrics data is loading THEN the Dashboard SHALL display skeleton loading states for each metric card

### Requirement 2

**User Story:** As a user, I want to view my subjects queue, so that I can see what topics are pending for article generation.

#### Acceptance Criteria

1. WHEN a user views the topics queue section THEN the Dashboard SHALL display subjects in numbered order
2. WHEN the topics queue contains more than 8 subjects THEN the Dashboard SHALL display the first 8 subjects with a count of remaining items
3. WHEN the topics queue is empty THEN the Dashboard SHALL display an empty state with a call-to-action to add topics
4. WHEN a user clicks the add topics button THEN the Dashboard SHALL navigate to the subjects management page

### Requirement 3

**User Story:** As a user, I want to view my recent articles, so that I can track the status of my generated content.

#### Acceptance Criteria

1. WHEN a user views the recent articles section THEN the Dashboard SHALL display up to 5 most recent articles
2. WHEN displaying an article THEN the Dashboard SHALL show the article subject, generation date, and status badge
3. WHEN an article has status "sent" THEN the Dashboard SHALL display a default badge style
4. WHEN an article has status "pending" THEN the Dashboard SHALL display a secondary badge style
5. WHEN an article has status "draft" THEN the Dashboard SHALL display an outline badge style

### Requirement 4

**User Story:** As a user, I want to see my delivery settings at a glance, so that I can verify my configuration is correct.

#### Acceptance Criteria

1. WHEN a user views the delivery settings card THEN the Dashboard SHALL display the user email address
2. WHEN a user views the delivery settings card THEN the Dashboard SHALL display the user display name
3. WHEN a user views the delivery settings card THEN the Dashboard SHALL display selected delivery days as badges
4. WHEN all 7 days are selected THEN the Dashboard SHALL display a single "Everyday" badge instead of individual day badges
5. WHEN a user views the delivery settings card THEN the Dashboard SHALL display the preferred language with flag emoji

### Requirement 5

**User Story:** As a user, I want to see my writing style configuration, so that I can understand how my AI model is trained.

#### Acceptance Criteria

1. WHEN a user views the writing style card THEN the Dashboard SHALL display the count of style samples used for training
2. WHEN a user views the writing style card THEN the Dashboard SHALL display descriptive text about the AI training
3. WHEN a user clicks the update button THEN the Dashboard SHALL navigate to the style samples management page

### Requirement 6

**User Story:** As a user, I want the dashboard to handle errors gracefully, so that I understand when something goes wrong.

#### Acceptance Criteria

1. IF the API request fails THEN the Dashboard SHALL display an error message with the failure reason
2. IF the user is not authenticated THEN the Dashboard SHALL redirect to the login page
3. WHEN an error occurs THEN the Dashboard SHALL log the error details to the console for debugging

### Requirement 7

**User Story:** As a user, I want the dashboard data to reflect current state, so that I see accurate information.

#### Acceptance Criteria

1. WHEN the dashboard component mounts THEN the Dashboard SHALL fetch fresh data from the API
2. WHEN the user object changes THEN the Dashboard SHALL refetch dashboard data
3. WHEN data is successfully fetched THEN the Dashboard SHALL update all displayed metrics and content
